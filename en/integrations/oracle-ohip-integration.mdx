---
title: "Oracle OHIP Integration"
description: "How to use Oracle OPERA Cloud/OHIP with DialogShift for booking agents and Journey Messaging."
---

The Oracle OHIP integration connects DialogShift to Oracle OPERA Cloud through the Oracle Hospitality Integration Platform. Chat AI and Phone AI can show availability and rates; Journey Messaging can use PMS data from Oracle OHIP. Checkout through DialogShift is not available.

<Note>
  **Closed Beta Rollout:** This integration is not generally released for production use yet. Activation is only available in coordination with Customer Success.
</Note>

## Prerequisites

- Your hotel uses Oracle OPERA Cloud/OHIP.
- Customer Success has gateway URL, app key, OAuth credentials and the hotel ID.

## Activate the integration

There is no Oracle OHIP integration page in the DialogShift app. Customer Success activates the engine and PMS connection for your hotel.

1. Provide the OHIP credentials to Customer Success.
2. Customer Success activates Oracle OHIP as a booking engine or Journey Messaging integration.
3. If you use booking agents, check availability and rates in the DialogShift Availability Analyzer.

## Booking agents

| Topic | Behavior |
|:------|:---------|
| Company code / coupons / discount codes | Not supported. |
| Minimum stay in the dialog | Not explicitly communicated by DialogShift. |
| Checkout workflow | No DialogShift checkout. |
| Payment link in chat | Not supported. |
| Payment link by phone | Not supported. |
| Cancellation of unpaid bookings | Not applicable because DialogShift does not create an Oracle OHIP booking. |
| Payment variants | Not bookable through DialogShift. |
| Offer/quote function | No. |
| Required guest data | None in DialogShift checkout because there is no DialogShift checkout. |

## Booking agent configuration

**Gateway URL.** The gateway URL is configurable so DialogShift queries the correct Oracle OHIP environment. Customer Success enters the URL that belongs to your OPERA Cloud instance.

**App key.** The app key identifies the enabled OHIP application. Provide Customer Success with the key that was enabled for the DialogShift connection.

**OAuth credentials.** OAuth credentials are required so DialogShift can retrieve availability and rates with authorization. Use only credentials that are enabled for the interface and the correct hotel.

**Hotel ID.** The hotel ID controls which property Oracle OHIP returns data for. After setup, verify in the Availability Analyzer that the correct rooms and rates appear.

## Journey Messaging

| Topic | Behavior |
|:------|:---------|
| Supported | Beta. Oracle OHIP can provide booking and guest data for Journey Messaging. |
| Data flow | DialogShift reads reservations from OHIP regularly. |
| Updates | Polling, about every 15 minutes. Changes are therefore not visible immediately. |
| Activation | Customer Success sets up the PMS connection. |
| First sync | The first sync can be started manually or in a controlled way so existing reservations are imported cleanly. |

## Configuration

Oracle OHIP requires gateway URL, app key, OAuth credentials and the hotel ID.
