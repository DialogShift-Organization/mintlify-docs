---
title: "apaleo Integration"
description: "How to use apaleo with DialogShift for booking agents and Journey Messaging."
---

The apaleo integration connects DialogShift to your apaleo property. It can power both booking agents and Journey Messaging.

## Prerequisites

- Your hotel uses apaleo.
- The apaleo integration is connected in the DialogShift app.
- The correct property is mapped to your DialogShift client.

## Activate the integration

Because there is an apaleo integration page in the DialogShift app, the hotel can activate the integration itself.

1. Connect apaleo in the DialogShift app via **Connect**.
2. Map the apaleo property to the correct DialogShift client.
3. If needed, select apaleo as the booking engine for the agent.

## Booking agents

| Topic | Behavior |
|:------|:---------|
| Company code / coupons / discount codes | Not supported. |
| Minimum stay in the dialog | Not explicitly communicated by DialogShift. |
| Checkout workflow | Direct booking in apaleo. For rates requiring a credit card or prepayment, DialogShift creates an apaleo/Adyen payment link afterwards. |
| Payment link in chat | The apaleo payment link is shown in chat. |
| Payment link by phone | When apaleo is enabled for the Phone Full Booking Agent, the apaleo payment link is sent by SMS/WhatsApp. |
| Cancellation of unpaid bookings | Yes. Unpaid payment-link bookings are automatically cancelled after currently 75 minutes. |
| Payment variants | No prepayment: direct confirmation. Credit-card guarantee and prepayment: payment/card details through the apaleo/Adyen link. |
| Offer/quote function | The apaleo offers API is used for availability and technical booking data; there is no separate offer link. |
| Required guest data | Email, first name and last name are required. Phone number is optional. |

## Journey Messaging

| Topic | Behavior |
|:------|:---------|
| Supported | Yes. The same apaleo connection can provide booking and guest data for Journey Messaging. |
| Data flow | apaleo sends reservation updates to DialogShift via webhooks. |
| Updates | Near real-time once the property is mapped correctly. |
| Activation | The apaleo connection and property mapping are shared. Journey Messaging flows and templates are configured separately in DialogShift. |
| Limitation | Booking-agent settings such as channel, rate plans and images are separate and do not affect Journey Messaging. |

## Configuration

In the DialogShift app, define which apaleo channels, rate plans and room images the booking agents use. For Journey Messaging, maintain flows and templates under **Journey Messaging**.
