---
title: "Samera Integration"
description: "How to use Samera with DialogShift for availability and checkout links in chat and on the phone."
---

The Samera integration connects DialogShift to your Samera booking engine. Chat AI and Phone AI can retrieve availability and rates and send a prepared Samera checkout link to the guest. Journey Messaging is not supported for Samera.

## Prerequisites

- Your hotel uses Samera.
- Customer Success knows the `oneIdentifier` of your Samera subdomain.

## Activate the integration

There is no Samera integration page in the DialogShift app. Customer Success activates the engine for your hotel.

1. Provide the Samera `oneIdentifier` to Customer Success.
2. Customer Success activates Samera as the booking engine.
3. Then enable the room categories and rates the agent may show.

## Booking agents

| Topic | Behavior |
|:------|:---------|
| Company code / coupons / discount codes | Not supported. |
| Minimum stay in the dialog | Yes. If Samera reports a minimum stay, the agent communicates the minimum nights. |
| Checkout workflow | Checkout via link. DialogShift creates a Samera cart and sends the checkout link. |
| Payment link in chat | The Samera checkout link is shown in chat. |
| Payment link by phone | The Samera checkout link is sent by SMS/WhatsApp. |
| Cancellation of unpaid bookings | No. |
| Payment variants | Payment and card guarantee are handled in the Samera checkout. |
| Offer/quote function | DialogShift adds offers to a Samera cart; there is no separate offer link. |
| Required guest data | None in the dialog. Already collected contact details can optionally prefill the Samera checkout. |

## Booking agent configuration

**Rooms.** Rooms are configurable so the agent only adds categories to the Samera cart that should be sold in the dialog. Enable only public categories that can also be booked correctly in the Samera checkout.

**Rates.** Rates are configurable because the guest completes payment and card guarantee in the Samera checkout. Enable only rates whose conditions are understandable in the dialog and whose checkout works in Samera.

**Room images.** Room images are configurable so guests can compare suggested categories more easily in chat. Add a clear hero image for each enabled category and verify the display in the Availability Analyzer.

## Journey Messaging

Samera is not currently supported as a PMS integration for Journey Messaging. To run automated journey flows, you need a supported PMS integration.

## Configuration

Customer Success sets up the Samera `oneIdentifier`. You then maintain the booking-agent configuration for enabled rooms, rates and images.
