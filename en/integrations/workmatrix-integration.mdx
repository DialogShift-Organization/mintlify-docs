---
title: "Workmatrix Integration"
description: "How to use Workmatrix with DialogShift for availability and rates in chat and on the phone."
---

The Workmatrix integration connects DialogShift to Workmatrix for availability and rate lookup. Journey Messaging and checkout through DialogShift are not supported for Workmatrix.

## Prerequisites

- Your hotel uses Workmatrix.
- Customer Success has the Workmatrix credentials.

## Activate the integration

There is no Workmatrix integration page in the DialogShift app. Customer Success activates the engine for your hotel.

1. Provide the Workmatrix credentials to Customer Success.
2. Customer Success activates Workmatrix as the booking engine.
3. Then check availability and rates in the DialogShift Availability Analyzer.

## Booking agents

| Topic | Behavior |
|:------|:---------|
| Company code / coupons / discount codes | Not supported. |
| Minimum stay in the dialog | Not explicitly communicated by DialogShift. |
| Checkout workflow | No DialogShift checkout. |
| Payment link in chat | Not supported. |
| Payment link by phone | Not supported. |
| Cancellation of unpaid bookings | Not applicable because DialogShift does not create a Workmatrix booking. |
| Payment variants | Not bookable through DialogShift. |
| Offer/quote function | No. |
| Required guest data | None in DialogShift checkout because there is no DialogShift checkout. |

## Booking agent configuration

**API credentials.** Workmatrix credentials are configurable so DialogShift reads availability and rates from the correct Workmatrix instance. Customer Success sets up these credentials for your hotel.

**Room and rate enablement.** Rooms and rates are configurable so the agent only shows offers that should be sold in the dialog. Enable only public offers with clear descriptions and verify them in the Availability Analyzer.

## Journey Messaging

Workmatrix is not currently supported as a PMS integration for Journey Messaging. To run automated journey flows, you need a supported PMS integration.

## Configuration

Customer Success sets up the technical Workmatrix connection.
