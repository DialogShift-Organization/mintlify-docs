---
title: "Ennismore / 25hours Integration"
description: "How to use Ennismore or 25hours with DialogShift for availability and rates in chat and on the phone."
---

The Ennismore integration connects DialogShift to the Ennismore Filter API. Chat AI and Phone AI can show availability and rates. Journey Messaging and checkout through DialogShift are not supported for Ennismore.

## Prerequisites

- Your hotel belongs to a supported Ennismore/25hours configuration.
- Customer Success has the required hotel reference.

## Activate the integration

There is no Ennismore integration page in the DialogShift app. Customer Success activates the engine for your hotel.

1. Provide the Ennismore hotel reference to Customer Success.
2. Customer Success activates Ennismore as the booking engine.
3. Then check availability and rates in the DialogShift Availability Analyzer.

## Booking agents

| Topic | Behavior |
|:------|:---------|
| Company code / coupons / discount codes | Not supported. |
| Minimum stay in the dialog | Not explicitly communicated by DialogShift. |
| Checkout workflow | No DialogShift checkout. |
| Payment link in chat | Not supported. |
| Payment link by phone | Not supported. |
| Cancellation of unpaid bookings | Not applicable because DialogShift does not create an Ennismore booking. |
| Payment variants | Deposit or credit-card information can be visible, but it is not bookable through DialogShift. |
| Offer/quote function | No. |
| Required guest data | None in DialogShift checkout because there is no DialogShift checkout. |

## Booking agent configuration

**Brand.** The brand is configurable so DialogShift queries the correct Ennismore/25hours environment. Customer Success sets the right brand for your hotel.

**Hotel reference.** The hotel reference controls which property availability and rates are read for. Provide the correct reference to Customer Success and then verify in the Availability Analyzer that the expected offers appear.

## Journey Messaging

Ennismore is not currently supported as a PMS integration for Journey Messaging. To run automated journey flows, you need a supported PMS integration.

## Configuration

Ennismore requires the correct brand and hotel reference.
