---
title: "Casablanca Integration"
description: "How to use Casablanca with DialogShift for booking agents and Journey Messaging."
---

The Casablanca integration can connect DialogShift to the Casablanca Cloud REST API for availability and rates and can also provide PMS data for Journey Messaging. Checkout through DialogShift is not available.

<Note>
  **Closed Beta Rollout:** This integration is not generally released for production use yet. Activation is only available in coordination with Customer Success.
</Note>

## Prerequisites

- Your hotel uses Casablanca.
- Customer Success has the Casablanca credentials and the correct hotel/tenant configuration.

## Activate the integration

There is no Casablanca integration page in the DialogShift app. Customer Success activates the booking engine and PMS connection for your hotel.

1. Provide the Casablanca credentials to Customer Success.
2. Customer Success activates Casablanca as a booking engine or Journey Messaging integration.
3. If you use booking agents, check availability and rates in the DialogShift Availability Analyzer.

## Booking agents

| Topic | Behavior |
|:------|:---------|
| Company code / coupons / discount codes | Not supported. |
| Minimum stay in the dialog | Yes. Minimum stays or unavailable stay lengths are communicated in the dialog. |
| Checkout workflow | No DialogShift checkout. |
| Payment link in chat | Not supported. |
| Payment link by phone | Not supported. |
| Cancellation of unpaid bookings | Not applicable because DialogShift does not create a Casablanca booking. |
| Payment variants | Credit-card requirement can be visible as rate information, but it is not bookable through DialogShift. |
| Offer/quote function | Booking-offer data is used for availability and rates, not for a payment link. |
| Required guest data | None in DialogShift checkout because there is no DialogShift checkout. |

## Booking agent configuration

**Cloud REST credentials.** Cloud REST credentials are configurable because booking agents read availability and rates from the Casablanca Cloud REST API. Customer Success sets up the credentials and maps them to the correct hotel/tenant.

**Hotel/tenant configuration.** The hotel/tenant mapping ensures the agent only shows offers for the correct property. After setup, verify in the Availability Analyzer that the expected rooms and rates are returned.

## Journey Messaging

| Topic | Behavior |
|:------|:---------|
| Supported | Yes. Casablanca can provide booking and guest data for Journey Messaging. |
| Data flow | Casablanca pushes PMS changes to DialogShift via webhooks. |
| Updates | Webhook-based; updates are processed after the PMS update. |
| Activation | Customer Success sets up the PMS connection. |

## Configuration

For Journey Messaging, Customer Success sets up the PMS connection separately.
