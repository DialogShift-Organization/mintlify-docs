---
title: "HNS Integration"
description: "How to use HNS with DialogShift for availability, rates and checkout links in chat and on the phone."
---

The Hotel Net Solutions (HNS) integration connects DialogShift to your OnePageBooking engine. Chat AI and Phone AI can retrieve availability, prices and rates and send guests to the HNS checkout link. Journey Messaging is not supported for HNS.

## Prerequisites

- Your hotel uses OnePageBooking from HNS.
- The integration is commissioned with HNS (paid service at HNS).
- Your DialogShift Client ID (in the app under **Settings → Integration**).

## Activate the integration

Because there is an HNS integration page in the DialogShift app, the hotel can activate the integration itself once HNS has provided the credentials.

1. Commission the DialogShift integration with HNS.
2. HNS sends the API credentials to you or directly to DialogShift.
3. The credentials are entered on the HNS integration page in the DialogShift app.
4. DialogShift sends your Client ID to HNS.

## Booking agents

| Topic | Behavior |
|:------|:---------|
| Company code / coupons / discount codes | Not supported. |
| Minimum stay in the dialog | Not explicitly communicated by DialogShift. |
| Checkout workflow | Always an HNS OnePageBooking checkout link. The guest completes the booking on the HNS page. |
| Payment link in chat | The HNS checkout link is shown in chat. |
| Payment link by phone | The HNS checkout link is sent by SMS/WhatsApp. |
| Cancellation of unpaid bookings | No. |
| Payment variants | Payment and card guarantee are handled in the HNS checkout. |
| Offer/quote function | No. |
| Required guest data | None in the DialogShift dialog; the guest enters the data in the HNS checkout. |

## Booking agent configuration

**Room categories.** Room categories are configurable so the agent only offers categories that should be sold in chat and on the phone. Do not enable internal, non-public or hard-to-explain categories.

**Rates.** Rates are configurable so the agent only sends guests to suitable public offers in the HNS checkout. Enable only rates whose conditions can be explained clearly in the dialog and completed correctly on the HNS page.

**Images.** Images are configurable because they help guests choose a category in chat. Add a suitable image for every enabled room category and verify in the Availability Analyzer that it appears correctly.

## Journey Messaging

HNS is not currently supported as a PMS integration for Journey Messaging. To run automated journey flows, you need a supported PMS integration.

## Configuration

The technical HNS connection is maintained on the HNS integration page in the DialogShift app.
