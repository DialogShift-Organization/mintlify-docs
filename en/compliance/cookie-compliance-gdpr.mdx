DialogShift offers two approaches for integrating the chat widget into your website's cookie compliance setup.

## Option 1: Integration into Website Cookie Management

The chat widget is loaded by the website's consent manager only after the visitor has consented to functional or marketing cookies.

**Advantages:**
- Chat opening rate is measurable
- User behavior scoring is possible

**Disadvantage:** Visitors who decline cookie consent will not see the chat widget.

## Option 2: DialogShift Managed Consent

The chat widget loads initially without setting any cookies. When a visitor starts a live chat, DialogShift requests consent within the chat. No integration with the website consent manager is needed.

**Advantage:** All website visitors can use the chatbot regardless of their cookie settings.

**Disadvantages:**
- Chat opening rate cannot be measured before interaction
- User behavior scoring is not possible until explicit consent is given

## Comparison

| | Website Cookie Management | DialogShift Managed Consent |
|---|---|---|
| Cookie consent via | Website consent manager | Within the chat |
| Chat opening rate measurable | Yes | No |
| User behavior scoring | Yes (after consent) | No (until consent) |
| Reach | Reduced by consent rate | All visitors |
