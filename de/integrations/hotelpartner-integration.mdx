title: "HotelPartner Integration"

---

Die HotelPartner-Integration verbindet DialogShift mit deiner HotelPartner-Booking-Engine. Chat AI und Phone AI können Verfügbarkeiten und Raten abrufen und je nach Rate direkt buchen oder einen Angebotslink versenden. Journey Messaging wird für HotelPartner nicht unterstützt.

## Voraussetzungen

- Dein Hotel nutzt HotelPartner.
- Die HotelPartner-Zugangsdaten sind vorhanden.

## Integration aktivieren

Da es eine HotelPartner-Integrationsseite in der DialogShift App gibt, kann das Hotel die Integration selbst aktivieren.

1. Trage die HotelPartner-Zugangsdaten in der DialogShift App ein.
2. Aktiviere HotelPartner als Booking Engine für deinen DialogShift Client.
3. Gib die Zimmertypen und Raten frei, die der Agent verkaufen darf.

## Buchungsagenten

| Thema | Verhalten |
|:------|:----------|
| Company-Code / Coupons / Rabattcodes | Unterstützt, wenn der Code für dein Hotel freigegeben ist. |
| Mindestaufenthalt im Dialog | Nicht explizit durch DialogShift kommuniziert. |
| Checkout-Workflow | Hybrid: direkte Reservierung bei Raten ohne Kreditkartenpflicht, Angebotslink bei Raten mit Kreditkartenpflicht oder unbekanntem Zahlungsstatus. |
| Payment-Link im Chat | Der HotelPartner-Angebots- oder Bestätigungslink wird im Chat ausgegeben. |
| Payment-Link im Phone | Der HotelPartner-Link wird per SMS/WhatsApp versendet. |
| Stornierung unbezahlter Buchungen | Nein. |
| Zahlungsvarianten | Direktbuchung ohne Kreditkartenpflicht; Angebotslink bei Kreditkartenpflicht oder Vorauszahlung. |
| Angebots-/Quote-Funktion | Ja. Zahlungspflichtige Raten werden über die HotelPartner-Angebotsfunktion buchbar gemacht. |
| Benötigte Gastdaten | Vorname und Nachname sind Pflicht. E-Mail ist optional. Für Phone-Linkversand wird eine Mobilnummer benötigt. |

## Konfiguration Buchungsagenten

**Zimmer.** Zimmer sind konfigurierbar, damit der Agent nur Kategorien anbietet, die Gäste direkt im Dialog anfragen oder buchen sollen. Gib nur öffentliche, buchbare Kategorien frei und schließe interne oder nicht verkaufsfähige Kategorien aus.

**Raten.** Raten sind konfigurierbar, weil HotelPartner je nach Zahlungsstatus direkte Buchungen oder Angebotslinks nutzt. Gib nur Raten frei, deren Bedingungen im Dialog klar erklärbar sind.

**Company-Codes.** Company-Codes sind konfigurierbar, weil nicht jeder Code für jeden Gast sichtbar sein darf. Gib nur Codes frei, die ausdrücklich für die Nutzung im Dialog vorgesehen sind.

## Journey Messaging

HotelPartner wird aktuell nicht als PMS-Integration für Journey Messaging unterstützt. Für automatische Journey-Flows brauchst du eine unterstützte PMS-Integration.

## Konfiguration

Die technische HotelPartner-Verbindung wird in der DialogShift App gepflegt.
