---
title: "HNS Integration"
description: "Wie du HNS mit DialogShift für Verfügbarkeiten, Raten und Checkout-Links in Chat und Telefon nutzt."
---

Die Integration mit Hotel Net Solutions (HNS) verbindet DialogShift mit deiner OnePageBooking-Engine. Chat AI und Phone AI können Verfügbarkeiten, Preise und Raten abrufen und Gäste per Checkout-Link zur HNS-Buchungsstrecke führen. Journey Messaging wird für HNS nicht unterstützt.

## Voraussetzungen

- Dein Hotel nutzt OnePageBooking von HNS.
- Die Integration ist bei HNS beauftragt (kostenpflichtig bei HNS).
- Deine DialogShift Client-ID (in der App unter **Einstellungen → Integration**).

## Integration aktivieren

Da es eine HNS-Integrationsseite in der DialogShift App gibt, kann das Hotel die Integration selbst aktivieren, sobald HNS die Zugangsdaten bereitgestellt hat.

1. Beauftrage die DialogShift-Integration bei HNS.
2. HNS übermittelt die API-Credentials an dich oder direkt an DialogShift.
3. Die Credentials werden in der DialogShift App auf der HNS-Integrationsseite eingetragen.
4. DialogShift übermittelt deine Client-ID an HNS.

## Buchungsagenten

| Thema | Verhalten |
|:------|:----------|
| Company-Code / Coupons / Rabattcodes | Nicht unterstützt. |
| Mindestaufenthalt im Dialog | Nicht explizit durch DialogShift kommuniziert. |
| Checkout-Workflow | Immer Checkout-Link zu HNS OnePageBooking. Die Buchung wird auf der HNS-Seite abgeschlossen. |
| Payment-Link im Chat | Der HNS-Checkout-Link wird im Chat ausgegeben. |
| Payment-Link im Phone | Der HNS-Checkout-Link wird per SMS/WhatsApp versendet. |
| Stornierung unbezahlter Buchungen | Nein. |
| Zahlungsvarianten | Zahlung und Kartengarantie laufen im HNS-Checkout. |
| Angebots-/Quote-Funktion | Nein. |
| Benötigte Gastdaten | Im Dialog keine; die Daten werden im HNS-Checkout eingegeben. |

## Konfiguration Buchungsagenten

**Zimmertypen.** Zimmertypen sind konfigurierbar, damit der Agent nur Kategorien anbietet, die im Chat und Telefon verkauft werden sollen. Gib keine internen, nicht öffentlichen oder erklärungsbedürftigen Kategorien frei.

**Raten.** Raten sind konfigurierbar, damit der Agent nur passende öffentliche Angebote in den HNS-Checkout führt. Gib nur Raten frei, deren Bedingungen im Dialog klar erklärt werden können und die auf der HNS-Seite korrekt abschließbar sind.

**Bilder.** Bilder sind konfigurierbar, weil sie im Chat die Entscheidung für eine Kategorie unterstützen. Hinterlege pro freigegebenem Zimmertyp ein passendes Bild und prüfe im Availability Analyzer, ob es korrekt erscheint.

## Journey Messaging

HNS wird aktuell nicht als PMS-Integration für Journey Messaging unterstützt. Für automatische Journey-Flows brauchst du eine unterstützte PMS-Integration.

## Konfiguration

Die technische HNS-Anbindung wird über die HNS-Integrationsseite in der DialogShift App gepflegt.
