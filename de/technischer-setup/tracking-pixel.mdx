---
title: 'Tracking-Pixel'
description: 'Abgeschlossene Buchungen an DialogShift melden, damit Conversions dem Gast und der Konversation zugeordnet werden.'
---

Das Tracking-Pixel meldet abgeschlossene Buchungen an DialogShift. Du baust es auf der Buchungsbestätigungsseite ein – entweder als Pixel im Browser oder als direkten Aufruf des Endpoints von deinem Server.

## Endpoint

```
https://ibe.prod.co25.net/pixel?clientid=DEINE_CLIENT_ID&amount=234.56&currency=EUR&custid=GAST_ID
```

## Parameter

| Parameter | Pflicht | Beschreibung | Beispiel |
| --- | --- | --- | --- |
| `clientid` | Pflicht | Von DialogShift bereitgestellte Client-ID | `pro123` |
| `amount` | Pflicht | Buchungsbetrag, Punkt als Dezimaltrennzeichen, ohne Tausendertrennzeichen | `234.56` |
| `currency` | Empfohlen | Währungscode (ISO 4217); ohne Angabe wird `EUR` angenommen | `EUR` |
| `custid` | Empfohlen | Von DialogShift an die Buchungs-URL angehängte Gast-ID – unverändert zurückgeben | `c8f2a1` |
| `conversionType` | Optional | Standard ist `ibe` | `ibe` |
| `rooms` | Optional | Anzahl gebuchter Zimmer | `2` |
| `nights` | Optional | Anzahl Übernachtungen | `3` |
| `persons` | Optional | Anzahl Personen | `4` |
| `url` | Optional | URL der Buchungsbestätigungsseite | `https://...` |

## Gastzuordnung mit `custid`

DialogShift hängt beim Weiterleiten in die Buchungsstrecke einen `custid`-Parameter an die URL an (z. B. `…?…&custid=c8f2a1`). Lies diesen Wert aus der aufgerufenen URL aus, behalte ihn über die gesamte Buchungsstrecke und gib ihn im Pixel-Aufruf als `custid` zurück. Damit ordnet DialogShift die Buchung exakt diesem Gast zu.

Ohne `custid` erfolgt die Zuordnung über fingerprinting und ist deutlich ungenauer.

## Einbau

### Option A: Tracking-Pixel im Browser

Baue auf der Bestätigungsseite ein verstecktes Bild ein. Setze die Werte aus deiner Buchung ein:

```html
<script>
var pixel = document.createElement("img");
pixel.src = "https://ibe.prod.co25.net/pixel?clientid=DEINE_CLIENT_ID&amount=" + betrag + "&currency=" + waehrung + "&custid=" + custid;
pixel.style = "display:none;";
document.body.appendChild(pixel);
</script>
```

Du kannst dieses Snippet auch über den Google Tag Manager ausspielen.

### Option B: Endpoint direkt aufrufen

Rufe dieselbe URL serverseitig per `GET` auf, sobald die Buchung feststeht:

```
GET https://ibe.prod.co25.net/pixel?clientid=DEINE_CLIENT_ID&amount=234.56&currency=EUR&custid=c8f2a1
```

<Note>
Beim serverseitigen Aufruf kommt die IP deines Servers an, nicht die des Gastes. Gib deshalb immer `custid` mit – sonst kann DialogShift die Buchung nicht zuordnen.
</Note>
