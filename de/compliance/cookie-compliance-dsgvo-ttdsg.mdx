DialogShift bietet zwei Ansätze zur Einbindung des Chat-Widgets in das Cookie-Compliance-Setup deiner Website.

## Option 1: Einbindung in das Website Cookie Management

Das Chat-Widget wird vom Consent Manager der Website nur geladen, wenn der Besucher den funktionalen oder Marketing-Cookies zugestimmt hat.

**Vorteile:**
- Chat-Öffnungsrate ist messbar
- Verhaltensscoring der Nutzer ist möglich

**Nachteil:** Besucher, die die Cookie-Zustimmung ablehnen, sehen das Chat-Widget nicht.

## Option 2: DialogShift Managed Consent

Das Chat-Widget lädt zunächst ohne das Setzen von Cookies. Wenn ein Besucher einen Live-Chat startet, fragt DialogShift den Consent innerhalb des Chats ab. Eine Einbindung in den Website Consent Manager ist nicht erforderlich.

**Vorteil:** Alle Website-Besucher können den Chatbot nutzen, unabhängig von ihrer Cookie-Einstellung.

**Nachteile:**
- Die Chat-Öffnungsrate kann vor der Interaktion nicht gemessen werden
- Verhaltensscoring ist nicht möglich, solange keine explizite Zustimmung vorliegt

## Vergleich

| | Website Cookie Management | DialogShift Managed Consent |
|---|---|---|
| Cookie-Consent erfolgt über | Website Consent Manager | Im Chat selbst |
| Chat-Öffnungsrate messbar | Ja | Nein |
| Verhaltensscoring | Ja (nach Zustimmung) | Nein (bis zur Zustimmung) |
| Reichweite | Reduziert durch Ablehnungsrate | Alle Besucher |
